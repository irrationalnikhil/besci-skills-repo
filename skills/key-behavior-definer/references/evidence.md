# Evidence base — Key Behavior Definer

All citations peer-reviewed, post-2000, verified 2026-06-12.

## Core framework

- **Michie, S., van Stralen, M. M., & West, R. (2011).** The behaviour change wheel: A new method for characterising and designing behaviour change interventions. *Implementation Science*, 6, 42.
  - Relevance: behaviour *specification* ("define the behaviour in context: who, what, when, where, how often, with whom") is the documented entry step of the Behaviour Change Wheel. This skill operationalizes that step for non-specialists.
  - Type: framework/methodology paper, not an empirical effect estimate.

- **IL 3B methodology** (Identify Key Behavior → Reduce Barriers → Amplify Benefits). Practitioner methodology applied across 40+ behavior-change engagements; the Impact / Drop-off / Control prioritization grid and the "observable, measurable, uncomfortably specific" test originate here.
  - Type: practitioner methodology — *not* peer-reviewed; the skill keeps it because it is procedural scaffolding, not an empirical claim.

## Why specificity matters (supporting evidence)

- **Gollwitzer, P. M., & Sheeran, P. (2006).** Implementation intentions and goal achievement: A meta-analysis. *Advances in Experimental Social Psychology*, 38, 69–119. (Book-series chapter — flagged.)
  - Relevance: the WHO/WHAT/WHEN format borrows the logic of implementation intentions — specified when-where-how plans outperform vague goals (d ≈ 0.65 across 94 lab-heavy tests; field effects substantially smaller).

## Calibration notes for the skill's outputs

- The skill makes **no effect-size promises**. Defining a behavior well is necessary, not sufficient.
- WEIRD flag: canonical worked examples in the practitioner literature are US consumer tech. The skill's social-sector examples (WhatsApp practice activities, skin-to-skin initiation, advisory-line registration) are drawn from South Asian program contexts; teams elsewhere should localize.

## Known gaps

- No RCT directly tests "programs that define key behaviors via WHO/WHAT/WHEN outperform programs that don't." The evidence is framework-level + indirect. Be honest about this if asked.
